Xtream UI R22 Backend Administration API Manual Enterprise Billing Integration & Lifecycle Provisioning Specification This reference manual defines the complete backend API specification required to link an independent billing engine or storefront platform to an Xtream UI R22 / XUI core panel . This version incorporates commercial provisioning standards, secure credential parameters, and operational validation controls to ensure automated subscription processing, billing renewals, and failure handling maps perfectly to production requirements. 1. Global Integration Architecture All automated transactions must entirely bypass public streaming or player ports (such as standard port 80 or 8080). Direct your internal gateway scripts exclusively to the panel's central CMS manager administrative core port. Base Administrative Endpoint: http://YOUR_PANEL_IP:ADMIN_PORT/api.php Authentication Requirements: Every administrative payload must securely pass root or executive reseller verification flags inside the query arguments string: developer_username: The administrative dashboard gateway user credential. developer_password: The administrative dashboard gateway authentication password. 🔒 Security / Firewall Constraint: Xtream UI administrative CMS ports (commonly port 25500) must be firewalled via iptables or ufw on your core media server. Explicitly whitelist only the public IP address of your dedicated billing server platform to prevent unauthenticated access or remote execution risks. 2. Automated Account Lifecycle Provisioning Create a New Subscription Line (sub=create) Executed automatically by your billing engine's webhook handlers once a client invoice status transitions to Paid. This provisions a unique line in the database, applies simultaneous stream boundaries, and appends a fixed expiration date. http://YOUR_PANEL_IP:ADMIN_PORT/api.php?action=user&sub=create&developer_username=ADMIN_USER&developer_password=ADMIN_PASS&username=CLIENT_USER&password=CLIENT_PASS&max_connections=1&exp_date=TIMESTAMP&bouquet=BOUQUET_JSON_ARRAY Request Key Matrix Parameters: Key Parameter Data Type Description Production Best Practice username String Unique string line identifier. Strip client's email prefix and append a random 4-char suffix (e.g., john_a8k2). password String Stream validation access key. Generate a strict 10-character random alphanumeric string at checkout. max_connections Integer Simultaneous stream limit cap. Set dynamically based on the specific product package tier purchased. exp_date Epoch Unix Absolute time boundary of account. Calculate mathematically via time() stamp plus inventory plan duration. bouquet JSON Array URL-encoded integer list of Bouquet IDs. Extracted dynamically from your storefront package database mapping. Expected Payload Response: { &quot;status&quot;: &quot;success&quot;, &quot;message&quot;: &quot;User has been created successfully.&quot; } Modify or Renew an Active Line Profile (sub=edit) Invoked automatically for plan renewals, extensions, or product tier alterations. Do not generate a duplicate profile on renewal cycles; instead, update the existing database entry. http://YOUR_PANEL_IP:ADMIN_PORT/api.php?action=user&sub=edit&developer_username=ADMIN_USER&developer_password=ADMIN_PASS&username=EXISTING_CLIENT_USER&exp_date=NEW_TIMESTAMP&bouquet=NEW_BOUQUET_JSON_ARRAY ⚠️ Critical Overwrite Guard: Passing the bouquet parameter array completely flushes and replaces previous channel rows in the database. If your system allows distinct modular content add-ons or upsell packages, your middleware application must compute the combined mathematical array of existing and new Bouquet IDs before transmitting the edit payload. Disable / Suspend an Overdue Account (sub=disable) Triggers automatically when an invoice grace period lapses. This immediately terminates active client network streams and drops active TCP authentication challenges at the gateway. http://YOUR_PANEL_IP:ADMIN_PORT/api.php?action=user&sub=disable&developer_username=ADMIN_USER&developer_password=ADMIN_PASS&username=CLIENT_USER Expected Response: { &quot;status&quot;: &quot;success&quot;, &quot;message&quot;: &quot;User has been disabled successfully.&quot; } To Unsuspended / Reactivate: Issue an identical payload query string structure to the administrative URL but adjust the parameter flag to sub=enable . 3. Storefront Package & Inventory Alignment Query Master Bouquet Structure (action=bouquet&sub=get) Used to establish package mapping relations inside your central vendor control board. Your script calls this endpoint to extract active Bouquet Names and database IDs, eliminating manual data sync errors. http://YOUR_PANEL_IP:ADMIN_PORT/api.php?action=bouquet&sub=get&developer_username=ADMIN_USER&developer_password=ADMIN_PASS Expected Sync Return Payload: [ { &quot;id&quot;: &quot;1&quot;, &quot;bouquet_name&quot;: &quot;USA - Premium Live Entertainment&quot; }, { &quot;id&quot;: &quot;4&quot;, &quot;bouquet_name&quot;: &quot;UK - Sky Premium Sports Network&quot; }, { &quot;id&quot;: &quot;12&quot;, &quot;bouquet_name&quot;: &quot;VOD - 4K Cinema Vault&quot; } ] 4.
